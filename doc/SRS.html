1. Introduction

This document will layout the requirements for Project Jordan which is composed of two projects, which from here on will be referred to as My Basketball and NBAScrape. My Basketball is a web application that intends to be the goto web site for participants of a Fantasy Basketball league or an avid basketball fan inquiring about player statistics. The application will give the user the ability to view visual representations of chosen categories of all NBA players, which can be used to help users make managerial decisions. NBAScrape is that will parse websites such as http://stats.nba.com/stats, http://www.basketball-reference.com, https://basketballmonster.com, and http://espn.go.com/nba for NBA player information and images to stored in a shared database for My Basketball.

My Basketball is intended to be the home page for every hardcore NBA fan and Fantasy Basketball team owner. Users will have unique homepages customized to show up-to-date information about their team(s) and players on their watch-list. The goal and difference from other basketball reference websites is the emphasis on visualizing the statistics as graphs. For example, a simple chart of players sorted by PER for the day will be displayed as a neat bar graph, which will allow the user to process the information more quickly and soundly. The eventual goal of this web application is to turn it into a mobile app for iOS and Android platforms.

My Basketball will not be possible without the efforts of NBAScrape, which wil be an ongoing script that is used to fill the database that backs My Basketball. Some of the information hoped to be scraped includes general player information, daily statistics, player images, and buzzworthy information/links.

The main purpose of Project Jordan is to gain experience with the following concepts:
  • Full stack web applications
  • Mobile development
  • Web scraping
  • Object-oriented programming
  • Java
  • Python
  • HTML/CSS
  • JavaScript/jQuery
  • MongoDB
  • Regular Expressions
  • Concurrency
  • Testing with JUnit
  • REST and AJAX
  • Unix server environment
  • Documentation (Javadoc, Software Requirements Specification, Software Design Document)
  • Applying design patterns

1.1 Purpose

The purpose of this document is to precisely specify how the two programs of Project Jordan should look and operate. The intended audience for this document is all the members of the development team, from the statisticians, to the software engineer and even salespeople who will ultimately interface with the customers. This document serves as an agreement among all parties and a reference for how the application should ultimately be constructed. Upon completing the reading of this document, one should clearly visualize how it will look and operate as well as understand the rules of baseball and Fantasy Basketball.

1.2 Scope

This project has a broad scope, including the use of at least three different languages (for the sake of learning). My Basketball will hopefully have the domain name mybasketball.com which users can go to to access the web application. Project Jordan is currently planned to be completed by ultimately one person (Joon Lim).

2.3 User Characteristics

There are two defined end users for My Basketball.

Fantasy Basketball Team Owner
This is someone who is participating in one or more Fantasy Basketball leagues for the current season. There is something on the line (whether it be pride or cash) and the user actively seeks out any information that might give him/her the winning edge. The user will be inclined to make an account so that their team information and watch list are preserved and information and news about each player can be seen right away on the home page.

Avid NBA Fan
This is the fan who loves to watch basketball. They are not necessarily obsessed, but they care enough about the sport to look up highlights and information almost daily. They will be interested to see how players they like are doing in comparison to other players. Their main reason for coming to the site would be to view the previous night's top 10/5 plays or top performers list/graph.

The one type of user for NBAScrape are those developing My Basketball, for all the data to be used in the web application will come from NBAScrape.
